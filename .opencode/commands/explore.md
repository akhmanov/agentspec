# Workspace Explore Mode

Explore the task before formal lifecycle commitment for: $ARGUMENTS

Reference: `.opencode/skills/workspace-explore/SKILL.md`.

1. Restate the problem or question and whether this is pre-task exploration or an in-flight side-loop.
2. Explore the relevant code, docs, history, and topology broadly before narrowing the answer.
3. Clarify the user need, constraints, and trust boundaries that matter.
4. Assess whether the task is worth doing now, whether there is a simpler framing, and what could make the system worse.
5. Compare 2-3 viable directions and recommend one, including the option to defer or do nothing.
6. End with the next transition: stay exploratory, create or reuse a bead and go to `/spec`, or bounce back to `/spec` or `/plan` if execution assumptions broke.
